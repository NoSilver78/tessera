## Positive Beobachtungen
- **DENY ist nicht per Ack freischaltbar.** Reihenfolge `surface_veto → classification → ack → default` ist lückenlos; Ack wird nur im Default-UNKNOWN-Pfad erreicht (Tabellen-Treffer returnen vorher). DENY+passender Ack bleibt DENY (gegen echte Quelle + `test_ack_does_not_override_deny` verifiziert).
- **Trust-Anchor content_hash** ist über sortierte Quell-/Config-Dateien gebildet **und** an relative POSIX-Dateinamen gebunden (gate Z. 156–163) → path-swap erzeugt verschiedene Digests; wird immer frisch von Platte berechnet, nie aus dem Loader-Cache → stale-cached Inhalt kann nie matchen.
- **Default fail-closed:** jeder Miss + Loader-only-Komponente (content_hash `None`) → `UNKNOWN_BLOCK_ENFORCE`. Cross-Domain-Hash-Reuse leakt keinen Ack (domain-keyed).
- **Echte Dormancy/Read-Only:** verifiziert — keinerlei D9-Referenz außerhalb der zwei D9-Module, `__init__.py` importiert/ruft das Gate nicht, nur `read_bytes`/`read_text`/`rglob` — kein Store-Write, kein `hass.data`-Mutieren, keine Service-/Panel-Registrierung, keine Bus-Events.
- **Deterministischer Output** (sortierte Komponenten-Iteration, sortierte Surface-Reasons; byte-identisches JSON bei Doppellauf laut Reviewer).
- Die Reviewer-Mutationen (a)–(d) sind **empirisch sauber gekillt** (Hard-Veto, evidence_type-Gate, Ack-vor-Klassifikation, content_hash-Bindung im Ack-Pfad) — die Suite ist in ihrem Kern substanziell, nicht vacuous.
